# Идеи для следующих итераций

Короткий список того, что может упростить жизнь пользователя и админа — без раздувания интерфейса.

## Пользователь

- **Статус подписки** — срок действия и остаток трафика из XUI прямо на главной (один запрос `/user/me` + данные клиента).
- **Повтор оплаты** — кнопка «Оплатить» у pending-инвойса, если `confirmation_url` ещё актуален.
- **Уведомление об оплате** — после возврата с `return_url` автоматически обновлять список инвойсов.

## Админ

- **Список пользователей** — `GET /admin/users` с пагинацией вместо поиска только по ID.
- **Профиль пользователя** — на карточке: роль, инвойсы, быстрые действия (токен, XUI, удаление).
- **Создание платежа за пользователя** — `POST /admin/invoices` с `user_id` для ручной выставки счёта.
- **Лог последней проверки инвойсов** — когда запускали `/admin/invoices/check` и сколько активировано.

## Общее

- **Публичный health** — лёгкий `/api/health` без авторизации (для мониторинга).
- **Настройка суммы платежа** — вынести дефолт в `.env` (`APP__DEFAULT_INVOICE_AMOUNT`).
- **Тёмная тема** — переключатель в шапке, если понадобится.

## Не делать сейчас

- Полноценная админ-панель с таблицами и фильтрами.
- WebSocket и push-уведомления.
- Многостраничный роутинг — текущий SPA на одной странице достаточен.
